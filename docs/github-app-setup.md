# GitHub App Setup: XyphyX Bot

This document explains how to set up the **XyphyX Bot** GitHub App so that agent-created PRs appear under a bot identity rather than the board user's account. This allows the board user to review and approve agent PRs.

## Why This Is Needed

All PRs created by agents currently appear as authored by the board user (`carlarmstrong2`) because `gh` CLI and `git` are authenticated with the board user's credentials. GitHub does not allow PR authors to approve their own PRs.

---

## One-Time Setup (Board Action Required)

### Step 1: Create the GitHub App

1. Go to: `https://github.com/organizations/xyphyx/settings/apps/new`
2. Fill in:
   - **GitHub App name:** `XyphyX Bot`
   - **Homepage URL:** `https://github.com/xyphyx`
   - **Webhook:** Uncheck "Active"
3. Set **Repository permissions:**
   - Contents: **Read & Write**
   - Pull requests: **Read & Write**
   - Metadata: **Read** (auto-selected)
4. Under **Where can this GitHub App be installed?**, select **Only on this account**
5. Click **Create GitHub App**
6. Note the **App ID** shown at the top of the app settings page

### Step 2: Generate a Private Key

On the app settings page:

1. Scroll to **Private keys**
2. Click **Generate a private key**
3. Save the downloaded `.pem` file securely (e.g., `~/.config/xyphyx/xyphyx-bot.pem`)

### Step 3: Install the App on the Repo

1. On the app settings page, click **Install App** in the left sidebar
2. Click **Install** next to the `xyphyx` organization
3. Choose **Only select repositories** â†’ select `internal-project-template`
4. Click **Install**
5. After install, note the **Installation ID** from the URL:
   `https://github.com/organizations/xyphyx/settings/installations/**{INSTALLATION_ID}**`

### Step 4: Store Repo Secrets (for CI)

In `https://github.com/xyphyx/internal-project-template/settings/secrets/actions`:

| Secret name                    | Value                          |
| ------------------------------ | ------------------------------ |
| `XYPHYX_BOT_APP_ID`            | The numeric App ID from Step 1 |
| `XYPHYX_BOT_PRIVATE_KEY`       | Full contents of the `.pem` file |
| `XYPHYX_BOT_INSTALLATION_ID`   | The numeric Installation ID from Step 3 |

### Step 5: Configure Local Agent Environment

Add the following to your shell profile (e.g., `~/.bashrc`, `~/.zshrc`) or a `.env.local` file:

```bash
export XYPHYX_BOT_APP_ID=<your-app-id>
export XYPHYX_BOT_PRIVATE_KEY=~/.config/xyphyx/xyphyx-bot.pem
export XYPHYX_BOT_INSTALLATION_ID=<your-installation-id>
```

> **Note:** `XYPHYX_BOT_PRIVATE_KEY` can be either the file path to the `.pem` file or the raw PEM string.

## Usage (Agents)

### Creating a PR as the bot

Instead of `gh pr create`, use the wrapper script:

```bash
./scripts/agent-pr.sh \
  --title "feat: my feature" \
  --body "Description of changes" \
  --base main
```

All `gh pr create` flags are supported.

### Generating a token manually

```bash
BOT_TOKEN=$(node scripts/get-bot-token.mjs)
GH_TOKEN=$BOT_TOKEN gh pr create ...
```

---

## CI Workflows (Automated PRs)

For CI jobs that need to create PRs (e.g., automated dependency updates, release PRs), use `actions/create-github-app-token`:

```yaml
- uses: actions/create-github-app-token@v1
  id: app-token
  with:
    app-id: ${{ vars.XYPHYX_BOT_APP_ID }}
    private-key: ${{ secrets.XYPHYX_BOT_PRIVATE_KEY }}

- name: Create PR
  env:
    GH_TOKEN: ${{ steps.app-token.outputs.token }}
  run: gh pr create --title "..." --body "..." --base main
```

---

## Security Notes

- The `.pem` private key file must **never** be committed to the repository
- Rotate the private key periodically via the GitHub App settings page
- The installation token generated by `get-bot-token.mjs` expires after 1 hour
- Repo secrets are encrypted at rest by GitHub and only exposed to Actions runners
